## Fix #4: C-Style Cast Safety

### Status: ✅ COMPLETE

### Severity
**HIGH** - Unsafe casts hide type errors and cause undefined behavior

### Changes
**Replaced 5 C-style casts with safe casts:**

| File | Line | Before | After |
|------|------|--------|-------|
| set.hpp | 200 | `(set<T> *)p` | `dynamic_cast<set<T> *>(p)` + nullptr check |
| tuple.hpp | 201 | `(tuple2<T,T> *)p` | `dynamic_cast<tuple2<T,T> *>(p)` + nullptr check |
| tuple.hpp | 293 | `(tuple2<A,B> *)p` | `dynamic_cast<tuple2<A,B> *>(p)` + nullptr check |
| tuple.hpp | 300 | `(tuple2<A,B> *)p` | `dynamic_cast<tuple2<A,B> *>(p)` + nullptr check |
| exception.hpp | 34 | `(char*)malloc(...)` | `static_cast<char*>(malloc(...))` |

**Added 4 nullptr checks** after `dynamic_cast`

### Impact
- Type-safe comparisons
- No crashes on type mismatches
- Clear error handling
- Matches Python behavior

### Example
```cpp
// Before: Crash if wrong type
set<int> *s = (set<int> *)p;  // CRASH if p is a list
s->gcs.find(key);

// After: Safe handling
set<int> *s = dynamic_cast<set<int> *>(p);
if (!s) return False;  // No crash, returns False
```
